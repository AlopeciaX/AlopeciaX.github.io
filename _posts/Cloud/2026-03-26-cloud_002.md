# 애플리케이션 보안[과목3]

### SQL 인젝션

	SQL 코드를 삽입해 DB를 위,변조
		-> 'OR '1' ='1 이걸로 TRUE 값을 만들어서 인증 우회
	XSS - 저장형: 악성 스크립트를 게시판, 댓글에 저장
		-> 다른 사용자 방문 시 자동 실행
	XSS - 반사형: URL 파라미터에 스크립트 삽입
		-> 클릭하면 해당 브라우저에서 실행

### CSRF, 인증 취약점, 기타 웹

	CSRF: 로그인된 사용자가 정상 서버에 위조 요청 보냄, 피해자는 모름
	세션 하이재킹
	파일 업로드
	디렉터리 트래버설 등 존재

### HTTP 보안 헤더 & OWASP


### 시큐어 SDLC - 단계별

	1. 요구사항 분석
	2. 설계: 이 단계부터 시큐어 코딩 적용
	3. 구현: 시큐어 코딩 가이드 준수
	4. 테스트
	5. 배포, 운영

### 데이터베이스 보안 & 암호화

	DB 접근 통제 - 최소 권한
	DB 암호화
		1. API
		2. 플러그인
		3. TDE


### 소프트웨어 취약점 7대 유형

	1. 입력 데이터 검증, 표현: SQL 인젝션 - 프로그램 입력값에 대한 검증 누락
	2. 보안 기능: 인증, 인가, 암호화, 비밀번호 등 보안 기능의 부적절한 구현
	3. 시간, 상태: 병렬처리 시스템이나 연동 시스템에서 시간차를 이용한 공격
	4. 에러 처리: 에러 발생 -> 부적절한 메시지, 내부 시스템 정보 유출
	5. 코드 품질: 개발자 실수 -> 부적절한 자원 해제, NULL 포인터 역참조 등
	6. 캡슐화: 불충분한 캡슐화로 인한 정보 노출
	7. API 오용: 의도하지 않은, 안전하지 않은 API 사용
