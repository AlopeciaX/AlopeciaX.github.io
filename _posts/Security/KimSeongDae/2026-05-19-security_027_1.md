## STEP 2. GVM 설치

### 2-1. PostgreSQL + GVM 설치

```bash
apt install -y gvm postgresql
```

### 2-2. PostgreSQL 시작

```bash
systemctl start postgresql
```

### 2-3. 클러스터 버전 확인 (반드시 먼저 확인)

```bash
pg_lsclusters
```

```
# 출력 예시
Ver  Cluster  Port   Status
18   main     5432   online
```

> pg_upgradecluster는 실제 확인된 버전으로 실행  
> 이미 online 상태면 건너뛰어도 됨

```bash
# 예시: 17버전이 확인된 경우
pg_upgradecluster 17 main
```

### 2-4. collation 버전 갱신 (핵심 - 생략하면 DB 생성 실패)

```bash
sudo -u postgres psql -c "ALTER DATABASE template1 REFRESH COLLATION VERSION;"
sudo -u postgres psql -c "ALTER DATABASE postgres REFRESH COLLATION VERSION;"
```

```
# 정상 출력
template1 -> NOTICE: version has not changed  (이미 최신)
postgres  -> NOTICE: changing version from 2.41 to 2.42  (갱신 완료)
```

### 2-5. GVM DB 수동 생성

```bash
sudo runuser -u postgres -- /usr/share/gvm/create-postgresql-database
```

```
# 정상 출력
CREATE EXTENSION uuid-ossp
CREATE EXTENSION pgcrypto
CREATE EXTENSION pg-gvm
```

### 2-6. gvm-setup 실행 (10~30분 소요)

```bash
gvm-setup
```

> 완료 후 admin 비밀번호가 출력되면 반드시 저장  
> 비밀번호가 비어있으면 STEP 2-7에서 수동 설정

### 2-7. admin 비밀번호 수동 설정

```bash
systemctl stop gvmd
runuser -u _gvm -- gvmd --user=admin --new-password=원하는비밀번호
systemctl start gvmd
```

## STEP 4. 정보 수집

### 4-1. 타겟 IP 확인

```bash
nmap -sn 192.168.56.0/24
```

### 4-2. 전체 포트 스캔

```bash
nmap -sT -v -O -p0-65535 10.0.0.31 > metas3.txt
```

| 옵션 | 설명 |
|---|---|
| `-sT` | TCP Connect 스캔 |
| `-v` | 상세 출력 |
| `-O` | OS 탐지 |
| `-p0-65535` | 전체 포트 스캔 |
| `>` | 결과 파일 저장 |

### 4-3. 서비스 버전 상세 스캔

```bash
nmap -sV -sC -A 10.0.0.31
```

### 4-4. 추가 정보 수집

```bash
# SNMP 정보 수집
snmp-check 10.0.0.31
# community string이 "public" 이면 취약 -> 다른 이름으로 변경 권장

# FTP 취약점 스캔
nmap --script ftp-brute.nse 10.0.0.31

# SMB 취약점 스캔
nmap --script smb-vuln* 10.0.0.31

# 내 시스템 열린 포트 확인
ss -tulnp
netstat -na
```

### 🛡️ 방어 - Nmap / SNMP 스캔 대응

**Nmap 스캔 탐지 및 차단**

```bash
# Snort / Suricata 같은 IDS로 포트 스캔 패턴 탐지
# 방화벽에서 불필요한 포트 전부 차단
firewall-cmd --permanent --add-rich-rule='rule family=ipv4 source address="공격자IP" reject'
firewall-cmd --reload

# iptables로 포트 스캔 속도 제한 (초당 연결 수 제한)
iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 3 -j ACCEPT
iptables -A INPUT -p tcp --syn -j DROP
```

**SNMP 보안 강화**

```bash
# /etc/snmp/snmpd.conf 에서 community string 변경
# 기존: public -> 추측하기 어려운 문자열로 교체
vi /etc/snmp/snmpd.conf
```

```
# 변경 전 (취약)
rocommunity public

# 변경 후 (권장)
rocommunity MyS3cur3Str!ng
```

```bash
# 가능하면 SNMPv3(인증+암호화) 사용 권장
# SNMPv1/v2는 평문 전송이므로 내부망에서도 위험
systemctl restart snmpd
```

> **핵심 원칙**: 외부에서 접근할 필요 없는 서비스(SNMP, FTP 등)는  
> 방화벽에서 아예 막거나 내부 IP만 허용하는 것이 최선

### 🛡️ 방어 - 브루트포스 공격 대응

**① 계정 잠금 정책 설정 (5회 실패 시 잠금)**

```bash
# PAM 설정으로 로그인 실패 횟수 제한
vi /etc/security/faillock.conf
```

```
deny = 5          # 5회 실패 시 잠금
unlock_time = 300 # 300초(5분) 후 자동 해제
fail_interval = 60
```

```bash
# /etc/pam.d/sshd 또는 /etc/pam.d/system-auth 에 추가
auth required pam_faillock.so preauth
auth required pam_faillock.so authfail
```

```bash
# 잠긴 계정 확인 및 해제
faillock --user 계정명        # 잠금 상태 확인
faillock --user 계정명 --reset # 잠금 해제
```

**② fail2ban 설치 (자동 IP 차단)**

```bash
apt install -y fail2ban     # Debian/Kali
dnf install -y fail2ban     # Rocky/CentOS

vi /etc/fail2ban/jail.local
```

```ini
[sshd]
enabled  = true
port     = ssh
maxretry = 5        # 5회 실패 시
bantime  = 3600     # 1시간 차단
findtime = 600      # 10분 내 실패 횟수 기준

[vsftpd]
enabled  = true
port     = ftp
maxretry = 5
bantime  = 3600
findtime = 600
```

```bash
systemctl enable --now fail2ban

# 차단된 IP 확인
fail2ban-client status sshd
```

**③ SSH 키 인증으로 전환 (패스워드 로그인 비활성화)**

```bash
vi /etc/ssh/sshd_config
```

```
PasswordAuthentication no   # 패스워드 로그인 완전 차단
PermitRootLogin no          # root 직접 로그인 차단
MaxAuthTries 3              # 세션당 최대 시도 횟수
```

```bash
systemctl restart sshd
```

**④ 강력한 패스워드 정책 설정**

```bash
vi /etc/security/pwquality.conf
```

```
minlen = 12         # 최소 12자
dcredit = -1        # 숫자 최소 1개
ucredit = -1        # 대문자 최소 1개
lcredit = -1        # 소문자 최소 1개
ocredit = -1        # 특수문자 최소 1개
```

> **요약**: 브루트포스 방어의 핵심은  
> **① 잠금 정책** + **② 자동 IP 차단(fail2ban)** + **③ 키 인증** 세 가지 조합

## STEP 6. 익스플로잇 (Metasploit)

### 기본 흐름

```bash
msfconsole

search <서비스명>        # 모듈 검색

show options            # 옵션 확인
set RHOSTS 10.0.0.31    # 타겟 IP
set LHOST 10.0.0.x      # 내 IP
run                     # 실행
```

### Metasploitable3 Linux 주요 취약점

| 서비스 | 포트 | 검색 키워드 |
|---|---|---|
| FTP | 21 | `proftpd` |
| IRC | 6667 | `unreal` |
| Web | 80 | `drupal` |

### Metasploitable3 Windows 주요 취약점

| 서비스 | 포트 | 검색 키워드 |
|---|---|---|
| SMB | 445 | `ms17_010` |
| HFS | 8585 | `rejetto` |

### 🛡️ 방어 - 익스플로잇 대응

**① 보안 패치 최우선 적용**

```bash
# 대부분의 Metasploit 모듈은 이미 패치가 존재하는 취약점을 악용
# 정기적인 업데이트가 가장 효과적인 방어
dnf update -y       # Rocky/CentOS
apt upgrade -y      # Debian/Kali/Ubuntu
```

**② 불필요한 서비스 비활성화**

```bash
# 사용하지 않는 서비스는 아예 꺼두기
systemctl disable --now vsftpd    # FTP 미사용 시
systemctl disable --now telnet    # Telnet은 항상 차단
systemctl disable --now rpcbind   # 불필요한 RPC
```

**③ SMB / EternalBlue (MS17-010) 방어**

```bash
# Windows: SMBv1 비활성화
Set-SmbServerConfiguration -EnableSMB1Protocol $false

# 방화벽에서 445 포트 외부 접근 차단
firewall-cmd --permanent --remove-service=samba
firewall-cmd --reload
```

**④ RDP (BlueKeep) 방어**

```
- Windows 보안 업데이트 즉시 적용 (KB4499175 등)
- RDP 포트(3389)를 방화벽에서 허용된 IP만 접근 가능하도록 제한
- NLA(네트워크 수준 인증) 활성화
```

**⑤ 취약점 스캐너(GVM)로 선제적 점검**

```
GVM(OpenVAS)을 주기적으로 실행해 패치되지 않은 취약점을 먼저 발견
공격자보다 먼저 자신의 시스템을 스캔하는 것이 핵심
```

## 모의침투 절차 요약

```
STEP 1. 탐색          -> nmap -sn (네트워크 스캔)
STEP 2. 정보 수집     -> nmap -sV -sC -A (포트/서비스)
STEP 3. 취약점 분석   -> GVM / https://127.0.0.1:9392
STEP 4. 브루트포스    -> hydra / medusa
STEP 5. 익스플로잇    -> msfconsole
STEP 6. 권한 상승     -> 셸 획득 후 로컬 익스플로잇
```

## 🛡️ 방어 체계 요약

| 공격 기법 | 방어 방법 |
|---|---|
| Nmap 포트 스캔 | 방화벽으로 불필요한 포트 차단, IDS(Snort/Suricata) 도입 |
| SNMP 정보 수집 | community string을 `public` 외 다른 이름으로 변경, SNMPv3 사용 |
| FTP/SSH 브루트포스 | 5회 실패 시 계정 잠금(faillock), fail2ban으로 IP 자동 차단 |
| SSH 패스워드 공격 | 패스워드 로그인 비활성화, SSH 키 인증 전용으로 전환 |
| Metasploit 익스플로잇 | 정기 보안 패치, 불필요한 서비스 비활성화 |
| SMB(EternalBlue) | SMBv1 비활성화, 445 포트 외부 차단 |
| 웹 소스/파일 노출 | 디렉토리 리스팅 차단, 민감 파일 웹 루트 밖으로 이동 |
