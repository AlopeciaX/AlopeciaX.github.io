## 개요

Rocky Linux 환경에서 Docker 설치부터 MySQL/WordPress 컨테이너 구성, HAProxy 로드밸런싱, 네트워크 설정, 볼륨 마운트까지 실습한 내용을 정리합니다.

---

## 1. Docker 설치

### rocky 9-3 (이미지 로드)
```bash
dnf -y install dnf-plugins-core
dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
dnf -y install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
systemctl enable --now docker
docker load -i all.tar
```

### rocky 9-1 (이미지 저장 및 전송)
```bash
docker save -o all.tar alpine busybox httpd mysql:8.0 nginx rockylinux/rockylinux wordpress
scp all.tar root@10.0.0.13:/root/
```

## 2. MySQL + WordPress 컨테이너 구성

### rocky 9-1
```bash
# MySQL 컨테이너 실행
docker run -itd \
  -e MYSQL_ROOT_PASSWORD=It12345! \
  -e MYSQL_DATABASE=word \
  --name m1 mysql:8.0

# WordPress 컨테이너 2개 실행
docker run -itd -p 60080:80 \
  -e WORDPRESS_DB_HOST=172.17.0.2 \
  -e WORDPRESS_DB_USER=root \
  -e WORDPRESS_DB_PASSWORD=It12345! \
  -e WORDPRESS_DB_NAME=word \
  --name w1 wordpress

docker run -itd -p 60180:80 \
  -e WORDPRESS_DB_HOST=172.17.0.2 \
  -e WORDPRESS_DB_USER=root \
  -e WORDPRESS_DB_PASSWORD=It12345! \
  -e WORDPRESS_DB_NAME=word \
  --name w2 wordpress

# 커스텀 index.html 적용
docker cp index.html w1:/var/www/html/
docker cp index1.html w2:/var/www/html/index.html
```

## 3. HAProxy 로드밸런싱

> 포트 60080, 60180으로 직접 접근하기 번거로움 → HAProxy로 80포트 단일 진입점 구성

### rocky 9-4
```bash
vi /etc/haproxy/haproxy.cfg
systemctl start haproxy
firewall-cmd --add-port=80/tcp
```

### rocky 9-1, 9-2 (컨테이너 재구성)
```bash
docker rm -f $(docker ps -aq)
docker run -itd -p 65080:80 \
  -e WORDPRESS_DB_HOST=10.0.0.13 \
  -e WORDPRESS_DB_USER=root \
  -e WORDPRESS_DB_PASSWORD=It12345! \
  -e WORDPRESS_DB_NAME=word \
  --name w1 wordpress
```

### rocky 9-3 (MySQL host 네트워크 모드)
```bash
docker run -itd --net host \
  -e MYSQL_ROOT_PASSWORD=It12345! \
  -e MYSQL_DATABASE=word \
  --name m1 mysql:8.0
firewall-cmd --add-port=3306/tcp
```

**로드밸런서 동작 확인** (번외)
```bash
docker exec w1 sh -c "echo $HOSTNAME > /var/www/html/index.html"
```

## 4. Docker 네트워크

### --link 옵션 (단방향)
```bash
docker run -itd --name a1 alpine
docker run -itd --link a1 --name a2 alpine
docker exec a2 ping a1  # /etc/hosts에 a1이 등록되어 핑 가능
docker exec a2 cat /etc/hosts
```

> `--link` 옵션은 /etc/hosts에 상대 컨테이너를 등록하는 방식으로 단방향만 가능

### --add-host 옵션 (양방향)
```bash
docker run -itd --add-host a2:172.17.0.3 --name a1 alpine
docker run -itd --add-host a1:172.17.0.2 --name a2 alpine
docker exec a1 ping a2
docker exec a2 ping a1
```

## 5. Docker Volume

### 볼륨 생성 및 마운트
```bash
docker volume create www
docker run -itd -v www:/test --name a1 alpine

# 호스트에서 파일 생성 → 컨테이너에도 반영됨
touch /var/lib/docker/volumes/www/_data/test.txt
```

### httpd 볼륨 공유
```bash
docker run -itd -p 60080:80 -v www:/usr/local/apache2/htdocs --name h1 httpd
docker run -itd -p 60180:80 --volumes-from h1 --name h2 httpd
```

> `--volumes-from` 옵션으로 h1의 볼륨을 h2와 공유

## 6. 볼륨 영속성 실습

컨테이너 삭제 후 재생성해도 데이터가 유지되는지 확인

```bash
# 1단계 - 초기화
docker rm -f $(docker ps -aq)
docker volume rm sql www

# 2단계 - MySQL 생성 (볼륨 sql 마운트)
docker run -itd -v sql:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=It12345! \
  -e MYSQL_DATABASE=word \
  -e MYSQL_USER=jhjang \
  -e MYSQL_PASSWORD=It12345@ \
  --name m1 mysql:8.0

# 3단계 - WordPress 생성
docker run -itd -p 65080:80 \
  -e WORDPRESS_DB_HOST=172.17.0.2 \
  -e WORDPRESS_DB_USER=jhjang \
  -e WORDPRESS_DB_PASSWORD=It12345@ \
  -e WORDPRESS_DB_NAME=word \
  --name w1 wordpress

# 5단계 - m1 삭제
docker rm -f m1

# 6단계 - m2 재생성 (같은 볼륨 sql 연결)
docker run -itd -v sql:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=It12345! \
  -e MYSQL_DATABASE=word \
  -e MYSQL_USER=jhjang \
  -e MYSQL_PASSWORD=It12345@ \
  --name m2 mysql:8.0

# 7단계 - WordPress 재접속 후 기존 글 확인
```

> 컨테이너를 삭제해도 볼륨(sql)에 데이터가 남아 있어 m2로 재연결 시 기존 데이터 유지됨

## 핵심 정리

| 개념               | 설명                |
| ---------------- | ----------------- |
| `--link`         | 단방향 /etc/hosts 등록 |
| `--add-host`     | 양방향 수동 hosts 등록   |
| `-v 볼륨명:경로`      | named volume 마운트  |
| `--volumes-from` | 다른 컨테이너 볼륨 공유     |
| `--net host`     | 호스트 네트워크 직접 사용    |
