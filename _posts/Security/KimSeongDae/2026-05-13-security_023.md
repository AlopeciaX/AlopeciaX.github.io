# Active Directory

윈도우 서버는 3가지로 나뉜다.

Windows Server( AD를 통한 인증, Group Policy를 통한 관리 )
	
	Standalone( Local Logon )
		독립실행형. 다른 서버와 상관 관계가 없음
		
	Member Server ( Domain Logon or Local Logon )
		DC의 자원(User, Group, Computer)을 갖어다 쓰기 위해서 구성
		Application은 Member Server에 설치를 권장함.
		
	Domain Controller ( Domain Logon )
		Domain 전체를 제어하는 서버
			일정한 영역
			Domain Name
			고객과 직원 계정 및 패스워드 정보 저장
			Application 설치 불가 -> 일반 사용자들도 접근 가능하기 때문
			

Administrator는 다 같은 최고 권위자가 아니다.

DC는 중앙집중형 장치
Single Sign On 토큰으로 모든 연관 사이트 접속가능, 한번털리면 다 털림, 상향평준화된 보안장치

RODC: ReadOnlyDomainController

local -> site -> domain -> ou

Infrastructure Master: 서로 포레스트안에 존재하는 관계에서 a사용자에 대한 내용을 복제할 경우 a 수정시 다른 쪽에서는 변경되지않는다. 이런 문제를 해결하는게 이거다.

# Active Directory

## Windows Server 종류

Windows Server는 크게 3가지 형태로 나뉜다.

### Standalone Server
- Local Logon 방식 사용
- 독립 실행형 서버
- 다른 서버와 관계 없음

### Member Server
- Domain Logon 또는 Local Logon 가능
- Domain Controller의 자원(User, Group, Computer)을 사용하기 위해 구성
- 애플리케이션 설치를 권장

### Domain Controller (DC)
- Domain 전체를 제어하는 서버
- Domain 계정 및 패스워드 정보 저장
- 중앙 집중형 인증 관리
- 일반 사용자 접근 가능성이 있으므로 Application 설치 비권장

---

## Active Directory 특징

- 중앙 집중형 인증 구조
- Single Sign On(SSO) 사용
- 한 번 인증되면 연관 서비스 접근 가능
- 하나가 침해되면 전체 영향 가능성 존재
- 보안 수준이 상향 평준화됨

### RODC
Read Only Domain Controller

읽기 전용 Domain Controller

---

## AD 구조

```text
local -> site -> domain -> ou
```

### Infrastructure Master
다른 Forest 간 객체 복제 시 발생하는 정보 불일치 문제를 해결하는 역할

예시:
- A 사용자의 정보 수정
- 다른 Forest에 변경사항 미반영
- Infrastructure Master가 동기화 수행
