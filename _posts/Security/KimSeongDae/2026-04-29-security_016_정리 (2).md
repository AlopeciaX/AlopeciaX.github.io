## 1. 환경 구성

### GNS3 + VMware 주의사항
- GNS3와 VMware를 함께 사용하면 CPU 사용량이 **99%** 까지 올라갈 수 있음
- 대안: **VirtualBox** 사용 (네트워크 카드 종류만 7가지 제공)

### GNS3의 한계와 IOU
- GNS3는 **라우터**만 제공 → 스위치에 CLI로 접속해 설정 불가
- 이를 해결하기 위해 **IOU(IOS on Unix)** 사용
  - IOU는 **리눅스 가상머신**에서만 사용 가능

### 실습 VM 구성
| VM | 역할 |
|---|---|
| rocky9-1 | DHCP 서버 |
| rocky9-2 | vsftpd (FTP 서버) |

### netstat 명령어 (Windows)
```

netstat -na -p tcp   # TCP만 확인
```

|---|---|
| ESTAB | 연결 중 |

## 3. DHCP

### 동작 4단계
```
Discover → Offer → Request → ACK
```
- **Offer Message**: 지연시간이 느리면 받는 속도도 느려짐
- DHCP 임대 주소 파일의 내용 형식은 `vsftpd.conf` 형식과 유사함

### 서버 옵션 vs 범위 옵션
| 구분 | 개념 비유 | 우선순위 |
|---|---|---|
| 서버 옵션 | 전역 변수 | 낮음 |
| 범위 옵션 | 지역 변수 | 높음 (오버라이드) |

> 영향 범위가 **좁을수록** 우선순위가 더 높다.  
> Windows는 하위 단에 옵션이 없으면 동일한 옵션을 **상속**한다. (→ 공유폴더 만드는 법과 동일한 원리)

> DHCP 관리 콘솔 — 범위 옵션(003 라우터, 006 DNS 서버, 015 DNS 도메인) 및 예약 설정 화면
> ![](../../../assets/images/Pasted%20image%2020260429140307.png)

### GNS3 토폴로지 아이콘 변화
- `10.0.0.254` → `10.0.0.252` 로 변경되면서 아이콘이 **종이 모양 → 컴퓨터 모양**으로 바뀜

## 5. IIS FTP (Windows FTP 서버)

### IIS 설치
- **역할 서비스를 직접 체크**해야 함 (관리 콘솔은 필수 선택)
- 실행: `compmgmt.msc`
- IIS 설치 시 `inetpub` 폴더 자동 생성
  - ⚠️ 해커도 기본 경로를 알고 있으므로 **기본 폴더 사용 금지**

> IIS FTP 사이트 추가 화면 — 사이트 이름: `jhjang_main`, 경로: `C:\01_ftp` (기본 폴더 대신 별도 경로 사용 예시)
> ![](../../../assets/images/Pasted%20image%2020260429141731.png)

### 주요 설정 항목
| 항목 | 설명 |
|---|---|
| FTP 권한 부여 규칙 + FTP 인증 | 항상 함께 작업 |
| 디렉토리 검색 | MS-DOS는 리눅스용 → 윈도우 권한 인식 불가 → **UNIX**로 설정 |
| 로그 파일 시간 | 현지 시간 사용 안 함 → **세계 표준시(UTC)** 사용 |
| FTP 메시지(배너) | 변수 사용 시 `%username%` 형식, "메시지에서 사용자 변수 지원" 체크 필요 |
| FTP 방화벽 지원 | 전역 설정에서 패시브 모드 / 데이터 채널 포트 범위 설정 가능 |
| FTP 사용자 격리 | Linux의 `chroot`와 동일한 개념 → **전역 가상 디렉토리 사용** 시 `LocalUser` 폴더 반드시 생성, 사용자 이름 기준으로 실제 디렉토리 매핑 |

### 화이트리스트 vs 블랙리스트
- 항목 **삭제** 후 해당 항목을 **허용**으로 변경해야 정상 동작

> FTP IP 주소 및 도메인 제한 화면 — `10.0.0.101` 허용, 나머지 클라이언트는 **거부**로 설정 (화이트리스트 방식)
> ![](../../../assets/images/Pasted%20image%2020260429141844.png)

### 방화벽 포트 수동 추가
- Windows는 역할 설치 시 기본 포트만 자동 개방
- FTP는 예외적으로 **인바운드 규칙에 20, 21번 포트를 수동 추가** 필요

### 유용한 팁
- `Shift + 폴더 우클릭` → PowerShell 열기 메뉴 생성

## 6. FTP 클라이언트 명령어

### 파일 생성 (빈 파일)
| OS | 명령어 |
|---|---|
| Linux | `truncate`, `dd` |
| Windows | `fsutil file createnew ftp.txt 1024` |

### FTP 명령어 대응표
| 클라이언트 명령 | FTP 프로토콜 명령 | 설명 |
|---|---|---|
| `dir` | `LIST` | 서버 목록 |
| `get` | `RETR` | 다운로드 |
| `put` | `STOR` | 업로드 |
| `!dir` | — | 로컬 목록 확인 |
| `lcd` | — | 로컬 디렉토리 이동 |
| `cd` | — | 서버 디렉토리 이동 |

> Windows FTP 클라이언트에서는 `ftp>` 상태에서 **다른 사용자로 전환** 가능
