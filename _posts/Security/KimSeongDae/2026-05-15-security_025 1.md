## DHCP Failover

### 핵심 개념
- 주 DHCP / 보조 DHCP로 이중화 구성
- IP 범위를 7:3으로 분할 (주 70%, 보조 30%)
- 보조 DHCP에 100ms 지연 부여 → 주 서버 우선 응답 유도
- Failover Cluster 구성 시 50:50 로드밸런싱 가능
- 공유 저장소 없이 복제 방식으로 연결

### 실습
- mem1(주 DHCP), mem2(보조 DHCP) 구성
- mem1에서 고급 → 분할범위 설정
- mem2에서 보조 DHCP 활성화

### 트러블슈팅
- 보조 DHCP 활성화 안 하면 동작 안 됨 → 활성화 필수
- 클라이언트는 별다른 문제 없으면 이전 IP를 받아오려고 시도함

## IIS + DFS 복제

### 핵심 개념
- IIS 구성을 mem1, mem2 동일하게 유지
- DFS 복제로 mem1 콘텐츠를 mem2로 자동 복제
- 도메인 사용자 추가 시 `도메인\사용자명` 형식으로 입력

### 실습
- mem1에 `20_config` 폴더 생성
- DFS 관리 → 복제 설정
- DHCP 동적 업데이트: 보안된 항목만 체크 시 자동 업데이트

### 핵심 개념
- 적용 순서: 아래 → 위 (위쪽 설정이 아래를 덮어씀)
