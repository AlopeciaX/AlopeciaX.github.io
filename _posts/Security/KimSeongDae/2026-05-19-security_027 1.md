## STEP 2. GVM 설치

### 2-1. PostgreSQL + GVM 설치

```bash
apt install -y gvm postgresql
```

### 2-2. PostgreSQL 시작

```bash
systemctl start postgresql
```

### 2-3. 클러스터 버전 확인 (반드시 먼저 확인)

```bash
pg_lsclusters
```

```
# 출력 예시
Ver  Cluster  Port   Status
18   main     5432   online
```

> pg_upgradecluster는 실제 확인된 버전으로 실행  
> 이미 online 상태면 건너뛰어도 됨

### 2-4. collation 버전 갱신 (핵심 - 생략하면 DB 생성 실패)

```bash
sudo -u postgres psql -c "ALTER DATABASE template1 REFRESH COLLATION VERSION;"
sudo -u postgres psql -c "ALTER DATABASE postgres REFRESH COLLATION VERSION;"
```

```
# 정상 출력
template1 -> NOTICE: version has not changed  (이미 최신)
postgres  -> NOTICE: changing version from 2.41 to 2.42  (갱신 완료)
```

### 2-5. GVM DB 수동 생성

```bash
sudo runuser -u postgres -- /usr/share/gvm/create-postgresql-database
```

```
# 정상 출력
CREATE EXTENSION uuid-ossp
CREATE EXTENSION pgcrypto
CREATE EXTENSION pg-gvm
```

### 2-6. gvm-setup 실행 (10~30분 소요)

```bash
gvm-setup
```

> 완료 후 admin 비밀번호가 출력되면 반드시 저장  
> 비밀번호가 비어있으면 STEP 2-7에서 수동 설정

### 2-7. admin 비밀번호 수동 설정

```bash
systemctl stop gvmd
runuser -u _gvm -- gvmd --user=admin --new-password=원하는비밀번호
systemctl start gvmd
```

### 2-8. 설치 확인

```bash
gvm-check-setup
```

```
# 모든 항목 OK 확인
Step 1: OK - OpenVAS Scanner
Step 2: OK - GVMD Manager
Step 3: OK - Certificates
Step 4: OK - data
Step 5: OK - Postgresql DB and user
Step 6: OK - Greenbone Security Assistant
Step 7: OK - gvmd service / gsad service
Step 8: OK - nmap, ssh-keygen, nsis, xsltproc
Step 9: OK - greenbone-security-assistant
It seems like your GVM-25.04.0 installation is OK.
```

### 2-9. GVM 시작

```bash
gvm-start
```

### 2-10. CPU 안정될 때까지 대기

```bash
top
```

> 처음 실행 시 피드 업데이트로 CPU 80% 이상 올라감  
> 10~20분 대기 후 10% 이하로 내려오면 로그인

### 2-11. 브라우저 접속

```
https://127.0.0.1:9392
Username: admin
Password: 2-7에서 설정한 비밀번호
```

## STEP 4. 정보 수집

### 4-1. 타겟 IP 확인

```bash
nmap -sn 192.168.56.0/24
```

### 4-2. 전체 포트 스캔

```bash
nmap -sT -v -O -p0-65535 10.0.0.31 > metas3.txt
```

| 옵션 | 설명 |
|---|---|
| `-sT` | TCP Connect 스캔 |
| `-v` | 상세 출력 |
| `-O` | OS 탐지 |
| `-p0-65535` | 전체 포트 스캔 |
| `>` | 결과 파일 저장 |

### 4-3. 서비스 버전 상세 스캔

```bash
nmap -sV -sC -A 10.0.0.31
```

### 4-4. 추가 정보 수집

```bash
# SNMP 정보 수집
snmp-check 10.0.0.31
# community string이 "public" 이면 취약 -> 다른 이름으로 변경 권장

# FTP 취약점 스캔
nmap --script ftp-brute.nse 10.0.0.31

# SMB 취약점 스캔
nmap --script smb-vuln* 10.0.0.31

# 내 시스템 열린 포트 확인
ss -tulnp
netstat -na
```

## STEP 6. 익스플로잇 (Metasploit)

### 기본 흐름


## STEP 7. 실습 과제 (Rocky9-1)

```
1. Rocky9-1 초기화
2. vsftpd 설치
3. 계정: 알파벳 5글자로 생성
4. 패스워드: 숫자 + 알파벳 + 특수기호 조합 3글자
5. passive mode 설정
6. 포트포워딩 설정
7. 팀원끼리 모의침투 테스트 수행
```

### vsftpd 설치 및 설정

```bash
dnf install -y vsftpd
vi /etc/vsftpd/vsftpd.conf
```

```
anonymous_enable=NO
local_enable=YES
write_enable=YES
pasv_enable=YES
pasv_min_port=60000
pasv_max_port=60100
```

```bash
useradd abcde
passwd abcde

systemctl start vsftpd
systemctl enable vsftpd

firewall-cmd --permanent --add-service=ftp
firewall-cmd --reload
```

## 트러블슈팅

### GVM DB 생성 실패 시

```
오류: createdb: template database "template1" has a collation version mismatch
원인: PostgreSQL 업데이트 후 collation 버전 불일치
```

```bash
sudo -u postgres psql -c "ALTER DATABASE template1 REFRESH COLLATION VERSION;"
sudo -u postgres psql -c "ALTER DATABASE postgres REFRESH COLLATION VERSION;"
sudo runuser -u postgres -- /usr/share/gvm/create-postgresql-database
gvm-setup
```

### gsad 서비스 실행 실패 시

```
오류: Binding to port 9392 failed
원인: 이전 프로세스가 포트 점유 중
```

```bash
ss -tulnp | grep 9392
kill -9 <PID>
systemctl reset-failed gsad
systemctl restart gsad
```

### admin 비밀번호 분실 시

```bash
# 계정 확인
runuser -u _gvm -- gvmd --get-users

# 비밀번호 재설정
systemctl stop gvmd
runuser -u _gvm -- gvmd --user=admin --new-password=새비밀번호
systemctl start gvmd
```

### Kali 멈춤 / 검은 화면

```
원인: RAM 부족 또는 GVM 초기 실행 중 CPU 과부하
최소 RAM: 4GB / 권장: 8GB
```

```bash
# swap 추가
fallocate -l 4G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

## 모의침투 절차 요약

```
STEP 1. 탐색          -> nmap -sn (네트워크 스캔)
STEP 2. 정보 수집     -> nmap -sV -sC -A (포트/서비스)
STEP 3. 취약점 분석   -> GVM / https://127.0.0.1:9392
STEP 4. 브루트포스    -> hydra / medusa
STEP 5. 익스플로잇    -> msfconsole
STEP 6. 권한 상승     -> 셸 획득 후 로컬 익스플로잇
```
