	스텁: 하향식 통합 테스트에서 하위 모델 대체하는 가상 모듈
	드라이버: 상향식 통합 테스트에서 상위 모듈 역할하는 가상 모듈
	ISP: 인터페이스 세분화
	DIP: 의존 관계 역전 원칙
	서버리스 컴퓨팅: 서버직접관리X, 사용한 만큼만 비용 지불
	스머핑: 서비스 거부 공격 중 하나, ICMP Echo Request 패킷을 보내서 마비
	사용사례 다이어그램: 기능과 사용자간의 상호작용 표현
	RSA: 비대칭키 암호화 알고리즘, 큰 숫자의 소인수분해가 어렵다는 원리 이용한 알고리즘
	테스트 오라클: 테스트 결과가 참인지 판단하기 위해 정의된 참값
	chown: 소유주 변경 명령어

	스텁: 하향식 통합 테스트에서 하이 모듈을 대체하는 가상 모듈
	드라이버: 상향식 통합 테스트에서 상위 모듈을 대체하는 가상 모듈
	서버리스 컴퓨팅: 서버직접관리x, 사용량 과금 방식의 서비스 모델
	스머핑: 소스ip를 타겟으로 위조하여 icmp 패킷을 보내는 공격
	테스트 오라클: 테스트 결과가 참인지 판단하기 위해 미리 정의된 참값을 비교하는 기법
	기능적 응집도: 응집도의 가장 높은 단계
	내용 결합도: 결합도의 가장 낮은 단계, 한 모듈이 다른 모듈의 내부를 직접 참조하는 가장 나쁜 결합도
	DSA: 비대칭키 알고리즘, 이산대수의 어려움을 이용하는 알고리즘
	cat: 파일의 내용 출력 명령어
	지속성: 트랜잭션이 완료되면 영구적으로 보존되어야 한다는 성질

	AES: 대칭키 알고리즘, 미국 표준(NIST)
	DSA: 비대칭키 알고리즘, 이산대수 문제의 어려움을 이용한 알고리즘, 미국의 DDS 채택
	ECC: 타원 곡선 군, 이산대수 문제 이용한 알고리즘, RSA보다 짧은 키 길이
	공통 결합도: 두 모듈이 전역 데이터 참조, 내용 결합도 다음으로 나쁜 단계
	시간적 응집도: 특정 시간대에 함께 실행되어야 하는 기능을 모은 것
	조건 커버리지: 개별 조건식이 True/False를 한 번씩 수행하도록 하는 화이트박스 테스트 기준
	ps: 유닉스 환경에서 현재 시스템에서 실행 중인 프로세스들의 상태를 확인하는 명령어
	less: 유닉스 환경에서 파일의 내용을 페이지 단위로 나누어 화면에 출력하며 이전 페이지 되돌아가는 기능 포함
	트랜잭션: 데이터베이스에서 논리적 기능을 수행하는 하나의 작업 단위
	배치 다이어그램: 물리적인 노드와 통신 경로를 표현하는 다이어그램

	SRP: Single Reponsibility Principle, 단일 책임 원칙, 하나의 클래스는 하나의 책임만 지님
	OCP: Open-Closed Principle, 개방-폐쇄 원칙, 확장-open/수정-close, 기존걸 수정없이 추가 가능해야 함
	LSP: Liskov Substitution Principle, 리스코프 치환 원칙, 자식은 부모의 기능 온전하게 수행
	ISP: Interface Segregation Principle, 인터페이스 분리 원칙, 필요한 인터페이스만 정밀하게 분리 후 연결
	DIP: Dependency Inversion Principle, 의존 역전 원칙, 추상화에 의존, 상위모델->하위모델 의존 x

	삽입이상: 불필요한 데이터를 함께 삽입해야 하거나 삽입이 불가능한 현상
	제1 정규형: 모든 속성값이 원자 값이어야 함
	Git: 중앙 서버 없이 각 PC에 저장소를 복제하는 분산 버전 관리 시스템
	보안 컨테이너: DRM에서 콘텐츠를 안전하게 전달하기 위한 논리적 용기
	Maven: pom.xml을 사용하는 자바 빌드 도구의 명칭
	Hub & Spoke: 중앙에 메시지 통로를 두고 데이터를 전달하는 방사형 구조
	경계값 테스트: 테스트 데이터의 값들 중 경계값 선택하여 테스트하는 블랙박스 테스트 기법
	IoC: Inversion of Control, 제어의 역전, 프레임워크가 흐름을 주도하는 설계 원칙
	SQL Injection: 악의적인 SQL 구문을 삽입하여 데이터를 탈취하는 공격
	영향 분석: Impact Analysis, 모듈 변경 시 다른 모듈에 미치는 영향을 분석하는 작업
