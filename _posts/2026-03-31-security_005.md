# 실습순서

	1. 방화벽 장치를 pc와 연결
	2. putty로 방화벽에 접속
	   - root / secui00@!
	   - supermode / blue[mac주소끝4자리]max[날짜(일)]
	3. bluemax 접속 - admin / secui00!!
	4. 라이센스 미등록 시 등록
	5. 시간 및 인터페이스 설정
	6. 관리자 ip 설정(허용 ip설정) 및 admin외 추가로 쓸 관리자 등록(five5)
	7. 인터페이스 설정에서 eth2, eth3에 ip 주소 할당(ex)1.1.5.1/24, 2.2.5.1/24)
	   eth1에는 공인 ip주소를 할당, eth0에는 초기 실습용으로 주소 할당?
	8. 방화벽 외 pc에도 주소를 할당(네트워크에 맞게)
	9. 두 pc간 통신 확인 및 방화벽에 ssh 연결 후 dump로 icmp 확인
	10. 방화벽 정책으로 모두 허용인 any any any로 설정
	11. 정적 라우팅에서 상대방 게이트웨이 주소 허용(서로 다른 망에서의 통신)
	12. 객체로 외부망엔 공인ip, 내부망엔 pc1,pc2를 생성
	13. 이번엔 인터넷 연결 진행, 사설ip->공인ip를 위한 nat정책설정
	n:m port pool 방식으로 변환한 뒤 인터페이스를 인터넷랜이 연결된 eth1로 설정
	변경전 출발지 pc1,pc2로 한 뒤 변경후 출발지를 공인ip로 설정 이러면 외부의 ip주소를 할당받
	아서 인터넷 연결 가능

# 방화벽 초기 설정 실습 (SECUI MF2)

#방화벽 #SECUI #네트워크 #실습 #NAT #방화벽정책

---

## 📋 실습 개요

> SECUI MF2 방화벽 장비를 처음 도입했을 때의 초기 설정부터 인터넷 연결(NAT)까지 전 과정을 실습

---

## 1️⃣ 방화벽 접속

### 1.1 콘솔 접속 (PuTTY)

- PC와 방화벽 장치를 콘솔 케이블로 연결
- PuTTY 실행 후 Serial 접속

**계정 정보**

|계정|비밀번호|설명|
|---|---|---|
|`root`|`secui00@!`|일반 접속|
|`supermode`|`blue[MAC끝4자리]max[일(DD)]`|관리자 모드|

> [!example] 예시 MAC 주소 끝 4자리: `AB12`, 날짜: 15일 → 비밀번호: `blueAB12max15`

### 1.2 BlueMax 웹 GUI 접속

- 브라우저에서 방화벽 IP로 접속
- **ID:** `admin` / **PW:** `secui00!!`

---

## 2️⃣ 기본 설정

### 2.1 라이센스 등록

- BlueMax 접속 후 라이센스 미등록 시 먼저 등록 진행

### 2.2 시간 설정

- 시스템 시간 및 NTP 동기화 설정

### 2.3 관리자 설정

- **허용 IP 설정:** 방화벽에 접근 가능한 관리자 IP 지정
- **추가 관리자 계정 생성:** `five5` 계정 등록

---

## 3️⃣ 인터페이스 설정

### 3.1 인터페이스별 IP 할당

|인터페이스|용도|IP 예시|
|---|---|---|
|`eth0`|초기 실습용 (관리)|-|
|`eth1`|인터넷 (공인 IP)|공인 IP 할당|
|`eth2`|내부망 1|`1.1.5.1/24`|
|`eth3`|내부망 2|`2.2.5.1/24`|

### 3.2 PC IP 설정

- 각 PC에 방화벽 인터페이스와 동일 대역으로 IP 할당
    - PC1: `1.1.5.x/24` (게이트웨이: `1.1.5.1`)
    - PC2: `2.2.5.x/24` (게이트웨이: `2.2.5.1`)

---

## 4️⃣ 통신 테스트

### 4.1 PC 간 ping 확인

```bash
ping [상대방 PC IP]
```

### 4.2 tcpdump로 ICMP 패킷 확인

방화벽에 SSH 접속 후:

```bash
tcpdump -i eth2 icmp
# 또는
tcpdump -i eth3 icmp
```

---

## 5️⃣ 방화벽 정책 설정

### 5.1 전체 허용 정책 (초기 테스트용)

|출발지|목적지|서비스|액션|
|---|---|---|---|
|`Any`|`Any`|`Any`|`허용`|

> [!warning] 주의 `Any-Any-Any` 정책은 실습/테스트 목적이며, 실제 운영 환경에서는 사용 금지

---

## 6️⃣ 정적 라우팅 설정

- 서로 다른 망(eth2 ↔ eth3) 간 통신을 위한 라우팅 설정
- 상대방 게이트웨이 주소를 정적 라우팅에 추가

```
목적지 네트워크: 2.2.5.0/24
게이트웨이:     [상대방 방화벽 인터페이스 IP]
```

---

## 7️⃣ 객체 생성

|객체명|종류|값|
|---|---|---|
|공인IP|외부망 주소|할당된 공인 IP|
|PC1|내부망 호스트|`1.1.5.x`|
|PC2|내부망 호스트|`2.2.5.x`|

---

## 8️⃣ NAT 정책 설정 (인터넷 연결)

### 8.1 목적

> 사설 IP → 공인 IP 변환으로 인터넷 연결

### 8.2 설정 방식

- **변환 방식:** `n:m Port Pool` (NAPT)
- **적용 인터페이스:** `eth1` (인터넷 연결 구간)

### 8.3 NAT 정책 구성

|항목|값|
|---|---|
|변환 전 출발지|`PC1`, `PC2` (사설 IP)|
|변환 후 출발지|공인 IP|
|인터페이스|`eth1`|

### 8.4 동작 원리

```
[PC1: 1.1.5.x] ──┐
                  ├──→ [eth1: 공인IP] ──→ 인터넷
[PC2: 2.2.5.x] ──┘
       사설IP          공인IP로 변환
```
